Raspberry Pi

Background/History

Pi's are literally taking over the world. Since their creation in 2012, they have gained massive popularity due to their inexpensive nature (the latest Pi v.3 was released in February 2016 for USD35).
They are credit-card sized single board computers that were originally developed for teaching computer science in schools and developing countries - example here . However, they can be found streaming data from weather stations, used for home automaton and of course, hosting a website. They have now tired of this planet and even gone to the ISS. For a more sensible, and detailed history, go here.

Setting up an Apache web server

As you have already seen, Apache is a very useful web server application that allows interactive webpages to be served through linkage to scripting languages such as PHP.

To set up Apache on a Pi, follow the instructions below:

  • Install the apache2 package by typing the following into the Terminal:

    sudo apt-get install apache2 -y

  • By default, Apache will put a HTML file in the web folder. To test that the web server is working and the file is there, type:

    ls /var/www

    This should return index.html.
    If you are using another computer to test whether Apache is working on your Pi, you can simply type in the Pi's IP address.

  • Next, we will change the default web page. This is stored at: /var/www/index.html. Enter ls -l /var/www/index.html to look at the permissions on the file.
    This file will be owned by root. For a definition of what this means, look here . Normally we would change this so the file is not owned by root (so you can edit it directly) - but that's for another day. For now, we'll leave it as it is.

    Installing PHP

    Finally, we'll add that PHP joy back into your life by installing it on the Pi. Type in the following to install PHP:

    sudo apt-get install php5 libapache2-mod-php5 -y

    Now remove the index.html file by navigating to the directory where index.html is stored and typing:
